==== IDENTIFYING THE RELATIONSHIPS ====
USER -> TASK : ONE-TO-MANY
User can  have `MANY` task but, tasks belongs to 1 user.

CATEGORY -> TASK : ONE-TO-MANY
Each category can have `MANY` task but 1 tasks belongs to 1 category


USER -> TASK HISTORY : ONE-TO-MANY
1 user can have `MANY` task completion but task need to be completed by 1 user

USER -> AI_RECOMMENDATIONS : ONE-TO-MANY
1 User can have MANY` ai recommendations but each ai recommendation belongs to 1 user

TASKS -> AI_RECOMMENDATIONS : ONE-TO-MANY
Each task can have ~MANY~ ai recommandations , but each ai recommendation belongs to 1 task
